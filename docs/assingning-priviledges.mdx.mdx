## Roles & Privileges

When a new user is created, they are assigned a **role**.\
Each role comes with a set of **privileges** that determine what the user can see and do on the website.

---

## 1. Regular User Role

Basic account management such as updating profile information, changing passwords, and viewing personal details.

| **Role**     | **Description**                                                               | **Privileges**                                                                                                                                                                                                                                                                                   |
| ------------ | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Regular User | Can manage their own profile, change password, and view personal information. | `user:update:profile`, `user:update:password`, `user:view:id`, `enum:gender`, `enum:maritalStatus`, `enum:employmentType`, `enum:department`, `enum:documentType`, `enum:documentStatus`, `enum:exitType`, `enum:resignationStatus`, `enum:documentVisibility`, `enum:kycStatus`, `manager-type` |

---

## 2. Document Uploader Role

For users responsible for uploading and re-uploading documents, in addition to basic user permissions.

| **Role**          | **Description**                                                              | **Privileges**                                                                                                                      |
| ----------------- | ---------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| Document Uploader | Can upload and re-upload documents, along with all Regular User permissions. | `document:upload`, `document:reupload`, `document:view:id`, `document:user:view`, `sample:document:view`, `sample:document:view:id` |

---

## 3. Document Verification Role (Verification Officer)

For users responsible for verifying documents, updating document status, and uploading sample documents.

| **Role**                      | **Description**                                                                                                                      | **Privileges**                                                                                                                                                                                                          |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Document Verification Officer | Can view and verify documents, update document status, and upload/re-upload sample documents. Includes all Regular User permissions. | `document:view`, `document:view:id`, `document:user:view`, `document:user:documents`, `document:status:update`, `sample:document:upload`, `sample:document:reupload`, `sample:document:view`, `sample:document:view:id` |

---

## 4. Admin Role

For administrators responsible for managing user accounts, employee data, professional information, companies, and roles.

| **Role** | **Description**                                                                                                     | **Privileges**                                                                                                                                                                                                                                                                                                          |
| -------- | ------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Admin    | Can create and manage users, companies, roles, and professional information. Includes all Regular User permissions. | `user:create`, `user:reset-password`, `user:activate`, `user:view`, `user:admin:update`, `professional:information:create`, `professional:information:update`, `company:create`, `company:update`, `company:view`, `company:view:id`, `role:create`, `role:update`, `role:view`, `role:view:id`, `role:view:privileges` |

---

## 5. Super Admin Role

Reserved for system super administrators with complete access to manage and configure all parts of the system.

| **Role**    | **Description**                                                                                                                               | **Privileges**                                                                                                                                           |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Super Admin | Has full system access, including user uploads, company creation, role assignments, and privilege management. Includes all Admin permissions. | `privilege:create`, `privilege:assign`, `privilege:get:role`, `privilege:delete:role`, `admin:dashboard`, `upload:user`, `upload:company`, `upload:role` |

# Example Use Case

- A **Regular Employee** might only have:
  - `attendance:clock-in`, `attendance:clock-out`, `leave:create`, `leave:cancel`.
- An **HR Officer** might have:
  - All **Leave Module** privileges, `attendance:view:all`, `employee-evaluation:view`.
- A **Project Manager** might have:
  - All **Project & Task Module** privileges, plus `performance-evaluation:evaluator-update`.
